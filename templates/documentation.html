Core Philosophy

SecureAttic was developed to solve the "Master Password" dilemma. In traditional encryption, your security is only as strong as your ability to remember a complex string. However, passwords can be brute-forced, keylogged, or socially engineered.

"If the key doesn't exist in the digital world, it cannot be stolen by a digital thief."

By offloading the entropy of your encryption key to a physical hardware device, we create a multi-factor environment where unauthorized access requires physical theft of your hardware, not just a breach of your OS.

Installation & Setup

SecureAttic operates as a portable executable, designed for "Zero-Trace" deployment. It utilizes python-based webview architecture to minimize system footprint.

Step 01

Initialize the Environment

Run the SecureAttic.exe binary. The application will initialize the local registry files (`vault_targets.json`) and verify the presence of the validation token.

Step 02

The Binding Process

Insert your chosen USB drive. The application will scan all available drive letters. When you select a slot, it writes a hidden IndexerVolumeGuid token file to the root of the drive.

Step 03

Seed Generation

We generate a 12-word BIP-39 mnemonic phrase. Write this down. This is the only way to recover your data if your physical USB drive is lost or destroyed.

System Architecture

The SecureAttic engine operates at the user-mode level, utilizing the Python standard library to interact with the Windows filesystem and registry.

The "Vault" Layer

Dynamic file locking using atomic move operations. When locked, files are encrypted and renamed to stealth system drivers (e.g. win_driver_*.sys) to obfuscate their true nature.

Token Abstraction

Instead of traditional mounting, we verify the presence of the encrypted IndexerVolumeGuid master file on the target drive. This file contains the wrapped encryption key.

Encryption Standards

Every file within your Attic is encrypted using the Fernet symmetric encryption implementation. This guarantees that data cannot be read or modified without the key.

Cryptographic Primitives
  • Algorithm AES-128 (CBC Mode)
  • Integrity Check HMAC-SHA256
  • Key Generation SHA-256 Entropy Pooling
security_protocol.py 
// Step 1: Generate Entropy
entropy = os.urandom(16)

// Step 2: Create Master Key Hash
key_hash = hashlib.sha256(entropy).digest()

// Step 3: Initialize Fernet Cipher
fernet_key = base64.urlsafe_b64encode(key_hash)
cipher = Fernet(fernet_key)

// Step 4: Encrypt Data
encrypted_blob = cipher.encrypt(file_data)

Volume Binding

Most "security keys" just check for the volume label. SecureAttic binds to the drive by placing a cryptographically signed token file in the root directory.

Token Verification

We scan for IndexerVolumeGuid and SysRecovery.dat. These files contain the decryption keys needed to initialize the vault.

Stealth Attributes

The key files are marked with System, Hidden, and Read-Only attributes (0x01 | 0x02 | 0x04) via the Windows kernel, making them invisible to standard Explorer browsing.

Runtime Security

The "Sentry" Loop

The application spawns a background daemon thread known as the "Sentry." This thread operates independently of the main UI and polls the hardware bus state every 2 seconds.

  • Active Volume Polling

    Checks all mounted drive letters for the presence of the authorized IndexerVolumeGuid signature.

  • Instant Lock

    If the signature vanishes (USB pulled), the Sentry triggers the lockdown sequence, encrypting all target paths instantly.

  • Memory Scrubbing

    Upon locking, the clipboard is wiped via ctypes.windll.user32.EmptyClipboard and process working set size is reduced.

Troubleshooting

Key Not Recognized
Ensure the USB drive is accessible via a drive letter (e.g., E:\). The application requires read/write access to the root directory to verify the IndexerVolumeGuid file.
"Attic in Use" Error
This usually happens if a file inside the Attic is still open in another application (like Word or Photoshop). Close all associated apps before attempting to lock the Attic.
Slow Decryption Speeds
SecureAttic performs atomic moves and stealth attribute changes on every file. If you have thousands of small files, this can take time.

Extended FAQ

Does SecureAttic work on Mac/Linux?

Not currently. The system relies on ctypes.windll for attribute management. Mac/Linux support is planned.

What if my USB drive breaks?

If the drive fails physically, the key file is lost. You MUST use your Recovery Seed (provided at setup) to regenerate the encryption keys.

Are my files stored on the USB?

No. Your files stay on your PC. The USB drive only acts as the "physical key" to unlock them.
