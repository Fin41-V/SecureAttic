Core Philosophy

SecureAttic was developed to solve the "Master Password" dilemma. In traditional encryption, your security is only as strong as your ability to remember a complex string. However, passwords can be brute-forced, keylogged, or socially engineered.

"If the key doesn't exist in the digital world, it cannot be stolen by a digital thief."

By offloading the entropy of your encryption key to a physical hardware device and binding it to that specific drive's hardware serial number, we create a multi-factor environment where unauthorized access requires physical theft of your specific hardware, not just a cloned copy of the files.

Installation & Setup

SecureAttic operates as a portable executable with "Zero-Trace" design principles. It utilizes user-mode AppData storage to avoid requiring administrative privileges.

Step 01

Initialize the Environment

Run the SecureAttic.exe binary. The application initializes local configuration in %APPDATA%\SecureAttic, ensuring persistence without polluting the global registry.

Step 02

Hardware Binding

Insert your chosen USB drive. The application reads the Volume Serial Number via win32api and binds the encryption key to this specific hardware signature. A token file IndexerVolumeGuid is written to the drive root.

Step 03

Seed Generation

We generate a BIP-39 mnemonic phrase (English dictionary). Write this down. This allows key recovery if your hardware token is lost or destroyed, bypassing the hardware binding check.

System Architecture

The SecureAttic engine operates at the user-mode level, utilizing the Python standard library and `pywebview` for a lightweight UI.

The "Vault" Layer

Files are protected via atomic operations. When locked, data is encrypted and renamed with a .vault extension. A JSON header containing the original filename and timestamp is injected before the ciphertext to ensure perfect restoration.

Insomnia Mode

To prevent corruption during encryption operations, the system enforces ES_CONTINUOUS | ES_SYSTEM_REQUIRED, preventing Windows from sleeping while the application is active.

Encryption Standards

Every file within your Attic is encrypted using the Fernet symmetric encryption implementation. This guarantees that data cannot be read or modified without the key.

Cryptographic Primitives
  • Algorithm AES-128 (CBC Mode)
  • Integrity Check HMAC-SHA256
  • Key Generation SHA-256 Entropy Pooling

Volume Binding

SecureAttic implements robust hardware binding to prevent key cloning. A simple copy of the key file to another USB drive will not work.

Serial Hashing

The key file contains a payload in the format KEY_BYTES::SERIAL_HASH. We compute SHA256(Drive_Serial) at runtime and compare it against the stored hash. If they mismatch, the key is rejected.

Token Files

The system scans for IndexerVolumeGuid (Master) and SysRecovery.dat (Backup). These files are often hidden using System+Hidden attributes.

Runtime Security

The "Sentry" Loop

The application spawns a background daemon thread known as the "Sentry" (sentry_loop). This thread operates independently of the main UI and polls the hardware bus state every 2 seconds.

  • Auto-Lock Timer

    If unlocked for more than 15 minutes (900s), the Sentry automatically re-encrypts all data, clears the clipboard, and purges the key from RAM.

  • Instant Lock

    If the valid USB signature vanishes (drive pulled), the Sentry triggers the lockdown sequence immediately.

  • Duress Mode

    Entering specific "Panic Codes" (e.g. 9999) activates a fake interface state, populating the UI with dummy data while keeping real vaults encrypted.

Troubleshooting

Key Not Recognized
This can happen if you copied the key file to a new drive. SecureAttic binds to the hardware serial. You must use your Recovery Seed Phrase to regenerate a new key for the new drive.
"Attic in Use" / Access Denied
Files cannot be locked if they are open in another application. Ensure Word, Excel, or Photo viewers are closed before removing your key.
Antivirus False Positives
Because SecureAttic encrypts files rapidly, heuristics may flag it. We use the .vault extension to signal legitimate activity. Add the application folder to your AV exclusions if issues persist.

Extended FAQ

Does SecureAttic work on Mac/Linux?

Basic support exists. The code includes logic for macOS /Volumes scanning and Linux /media monitoring, as well as chflags hidden support for Mac.

Where is my config stored?

Configuration is stored in %APPDATA%/SecureAttic. This includes vault_targets.json and the encrypted local verification file.

Are my files stored on the USB?

No. Your files stay on your PC. The USB drive only acts as the "physical key" to unlock them.
