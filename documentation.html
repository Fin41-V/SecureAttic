Core Philosophy

SecureAttic was developed to solve the "Master Password" dilemma. In traditional encryption, your security is only as strong as your ability to remember a complex string. However, passwords can be brute-forced, keylogged, or socially engineered.

"If the key doesn't exist in the digital world, it cannot be stolen by a digital thief."

By offloading the entropy of your encryption key to a physical hardware device, we create a multi-factor environment where unauthorized access requires physical theft of your hardware, not just a breach of your OS.

System Architecture

The SecureAttic engine operates at the user-mode level but interfaces with the Windows.Devices.Enumeration and WMI (Windows Management Instrumentation) namespaces to pull unique controller data.

The "Vault" Layer

Virtual disk mapping that only appears when the Hardware UID is verified. This uses a proprietary file-system filter driver to intercept I/O requests.

Hardware Abstraction

Direct query to the USB Mass Storage Class (MSC) descriptors. We pull the iSerialNumber and bVndId directly from the device firmware.

Encryption Standards

Every file within your Attic is encrypted individually using a unique 96-bit nonce. We utilize AES-256-GCM to ensure that if a single bit of encrypted data is modified, the system will detect it and prevent decryption to protect against bit-rot or malicious injection.

Key Derivation Function (KDF)
  • Algorithm PBKDF2-HMAC-SHA512
  • Iterations 250,000
  • Salt Length 32 Bytes (Randomized)

// Step 1: Extract Device Descriptor
const deviceID = hardwareBus.getSerial("USB_PORT_0");

// Step 2: Mix with User Entropy
const seed = hash(deviceID + user_local_salt);

// Step 3: Derive Master Key (in volatile RAM)
const masterKey = argon2.derive(seed, {

// Step 4: Clear intermediate buffers

Troubleshooting

Device Not Recognized
Ensure the USB drive is using the GPT partition style. Some older MBR drives may not report their Serial Number correctly to the Windows API. Try a different USB port, preferably directly on the motherboard rather than a hub.
"Attic in Use" Error
This usually happens if a file inside the Attic is still open in another application (like Word or Photoshop). Close all associated apps before attempting to lock the Attic.
Slow Decryption Speeds
SecureAttic performs a health check on every file during decryption. If you have thousands of small files, this can take time. I recommend using the "Turbo Mode" in settings to skip non-critical integrity checks for large libraries.

Extended FAQ

Does SecureAttic work on Mac/Linux?

Not currently. The system relies on low-level Windows kernel calls for hardware binding. A Linux version is in the early research phase.

What if my USB drive breaks?

If the drive fails physically, the hardware signature is lost. You MUST use your Recovery Seed (provided at setup) or your pre-registered Backup Key to regain access.

Are my files stored on the USB?

No. Your files stay on your PC. The USB drive only acts as the "physical key" to unlock them.
